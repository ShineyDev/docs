Client

class github.Client(token, *, session, user_agent=None)

The base class for interaction with the API.

Parameters:

  • token (str) –

    A GitHub personal access token.

    See also

    • Creating a personal access token

    • View your personal access tokens

    Note

    Currently, fine-grained personal access tokens are not able to access the GraphQL API.

  • session (aiohttp.ClientSession) – A client session.

  • user_agent (str) –

    A user agent. Defaults to ShineyDev/github@VERSION:UUID.

    Note

    We request that you use your GitHub username or the name of your application for the User-Agent header value. This allows us to contact you should there be a problem.

await request(document, operation=None, /, **variables)

This function returns a coroutine.

Sends a request to GitHub’s GraphQL API.

Parameters:

  • document (str) –

    A GraphQL document.

    Tip

    If you haven’t already, you should learn GraphQL. You can also read GitHub’s GraphQL guides, find documentation in the GraphQL API reference, and use the GraphiQL Explorer to experiment with the API.

  • operation (str) – The name of the operation from the document to execute. Defaults to None, which implies that the document contains only one operation.

  • **variables – A mapping of GraphQL variables.

Raises:

  • ClientResponseHTTPError – Arbitrary HTTP error.

  • ClientResponseGraphQLError – Arbitrary GraphQL error.

Examples

>>> await client.request("{viewer{login}}")
{'viewer': {'login': 'nat'}}

>>> await client.request("query($login:String!){user(login:$login){name}}", login="nat")
{'user': {'name': 'Nat Friedman'}}
Return type:

dict
